# scripts/

Operational glue for the VPS. Nothing here is meant to run on the laptop.

## One-time VPS setup

Assumes a fresh Debian/Ubuntu VPS with `claude` already installed and authenticated for the user that will run the worker. (`claude auth login` is not part of bootstrap because per project constraint Claude is already authed.)

```sh
# 1. Copy this directory to the VPS
scp -r scripts/ vps:/tmp/llm-wiki-scripts/

# 2. Run the bootstrap as root
ssh vps
sudo bash /tmp/llm-wiki-scripts/bootstrap-vps.sh

# 3. Move scripts into place
sudo install -o llmwiki -g llmwiki -m 0755 /tmp/llm-wiki-scripts/run-*.sh /srv/llm-wiki/bin/
sudo install -o llmwiki -g llmwiki -m 0755 /tmp/llm-wiki-scripts/cron-lint.sh /srv/llm-wiki/bin/
sudo install -o llmwiki -g llmwiki -m 0755 /tmp/llm-wiki-scripts/smoke-curl.sh /srv/llm-wiki/bin/

# 4. Install systemd units
sudo install -o root -g root -m 0644 /tmp/llm-wiki-scripts/obsidian-headless.service /etc/systemd/system/
sudo install -o root -g root -m 0644 /tmp/llm-wiki-scripts/llm-wiki-api.service /etc/systemd/system/
sudo systemctl daemon-reload
```

## Obsidian Headless sync setup

```sh
# 5. Login to Obsidian Sync as the llmwiki user (interactive: email, password, MFA, e2e password)
sudo -u llmwiki ob login

# 6. List remote vaults to confirm names
sudo -u llmwiki ob sync-list-remote

# 7. Bind the remote vault to /srv/llm-wiki/vault
sudo -u llmwiki ob sync-setup
# When prompted, choose the right remote vault and point local path at /srv/llm-wiki/vault

# 8. Start continuous sync
sudo systemctl enable --now obsidian-headless.service
sudo systemctl status obsidian-headless.service
tail -f /srv/llm-wiki/log/obsidian.log
```

## Phase 1 manual ingest

After sync is working and you've dropped a source into `raw/inbox/` from your laptop:

```sh
sudo -u llmwiki /srv/llm-wiki/bin/run-ingest.sh raw/inbox/2026-04-07-test.md
sudo -u llmwiki /srv/llm-wiki/bin/run-query.sh "what does the test source say?"
sudo -u llmwiki /srv/llm-wiki/bin/run-query.sh "what does the test source say?" --save
sudo -u llmwiki /srv/llm-wiki/bin/run-lint.sh
```

## Phase 2 API deployment

After building the server (`cd server && npm ci && npm run build`), rsync `server/` to the VPS:

```sh
rsync -av --delete \
  --exclude node_modules --exclude test --exclude '*.test.ts' \
  server/ vps:/srv/llm-wiki/server/
ssh vps "cd /srv/llm-wiki/server && sudo -u llmwiki npm ci --omit=dev"
sudo systemctl enable --now llm-wiki-api.service
sudo systemctl status llm-wiki-api.service
journalctl -u llm-wiki-api -f
```

Smoke test from your laptop via SSH tunnel:

```sh
ssh -L 8080:localhost:8080 vps   # leave running
# in another terminal:
API_TOKEN=$(ssh vps "sudo cat /etc/llm-wiki/env" | grep ^API_TOKEN | cut -d= -f2)
API=http://127.0.0.1:8080 API_TOKEN="$API_TOKEN" bash scripts/smoke-curl.sh
```

## Phase 4 daily cron

```sh
sudo -u llmwiki crontab -e
# add:
0 3 * * * /srv/llm-wiki/bin/cron-lint.sh >> /srv/llm-wiki/log/cron-lint.log 2>&1
```

## Conventions

- **Humans only edit `raw/`.** Anything you change under `wiki/`, `index.md`, or `log.md` from the laptop will likely conflict with worker writes via Obsidian Sync.
- **One worker job at a time.** Both `run-*.sh` and the API server use the same lock file at `${LLMWIKI_LOCK:-/srv/llm-wiki/state/vault.lock}`. Trying to run two at once is rejected immediately, not queued.
- **Auth.** `API_TOKEN` is generated by `bootstrap-vps.sh` into `/etc/llm-wiki/env`. Read it with `sudo cat /etc/llm-wiki/env`. Rotate by editing the file and restarting `llm-wiki-api`.
